## 🏗️ Repository Architecture & Portfolio Assets
This sandbox environment is organized modularly to match the data taxonomy used in real-world critical infrastructure audits.

### 📄 1. Technical Writing & Compliance Documentation (`/technical-writing/`)
*   **System Security Plan (SSP):** An executive risk summary and technical report analyzing software supply-chain vulnerabilities in simulated substation routers.
*   **Incident Response Playbook:** A step-by-step Standard Operating Procedure (SOP) detailing physical air-gapping, host isolation, and perimeter defense protocols during an active operational network breach.
*   **Logical Access Control Policy:** A mock organizational policy enforcing the Principle of Least Privilege for physical server rooms and local engineering workstations.

### 💻 2. Defensive Automation Scripts (`/defensive-scripts/`)
*   **Asset Compliance Scanner:** An automated Python utility that audits active system network configurations and open TCP/UDP listening ports against an approved master configuration baseline.
*   **Perimeter Firewall Rules:** Mock command-line configuration scripts enforcing strict Electronic Security Perimeter (ESP) segmentation boundaries between corporate IT and industrial OT networks.
